Définitions

Qu'est ce qu'un système ?

Un système est une solution économique et performante répondant aux besoins d’un client tout en satisfaisant l’ensemble des parties prenantes.

Exemples

  • Une voiture

  • Un satellite

  • Un progiciel de gestion intégré

  • Un Environnement Numérique de Travail

Caractéristiques d'un système

  • Multi-fontionnel

  • Complexe

  • Assemblage de plusieurs constituants eux mêmes pouvant être des systèmes

Vue arborescente d'un système

Dans la suite du document nous désignerons un constituant d'un systèmes par l'une des expressions suivantes : "sous-système", "composant".

La vérification

Vérifier un système ou un constituant du système, c’est s’assurer qu’il est conforme à ses spécifications techniques.

La vérification est elle suffisante ?

Est ce que mon composant vérifié fonctionne toujours lorsqu’il est intégré ?

La validation

Valider un constituant du système c’est s’assurer qu’il est conforme aux exigences imposées par le niveau supérieur.

Valider un système c’est s’assurer qu’il fonctionne comme attendu par le client dans son environnement opérationnel.

Dans certains projets informatiques, la validation d'un système peut être associée à la notion de VABF : "Validation d'Aptitude au Bon Fonctionnement"

L'intégration

L’intégration est une opération systématique, réalisée en conformité à un plan d’intégration préétabli, d’assemblages successifs de composants dûment vérifiés et validés, en faisant à chaque pas les tests d’intégration et de non-régression planifiés, ainsi que, lors de l’obtention d’un constituant, sa vérification par rapport à ses exigences techniques et sa validation par rapport à ses exigences allouées par le niveau supérieur. In fine le système ainsi intégré est vérifié par rapport aux exigences système et validé par rapport à ses exigences initiales.

AFIS.

La qualification

Dans le cas d’un système unique, l’intégration correspond à la réalisation. Le système est alors intégré dans son environnement d’exploitation et de maintenance. Sa qualification opérationnelle est prononcée après vérification de son aptitude opérationnelle (incluant la vérification d’aptitude de son système de soutien), autorisant ainsi sa mise en exploitation.

AFIS.

Dans certains projets informatiques, la qualification opérationnelle d'un système peut être associée à la notion de VSR : "Vérification en Service Régulier"

Vérification et compilation

Un petit programme en Groovy 2.x

 
                    @groovy.transform.ToString 
                    class Voiture { 
                        Integer nbPorte = 3 
                        String couleur
                    } 
                     
                    def test() { 
                        new Voiture(couleur:15) 
                    } 
                    
                    test()

Le programme affiche le résultat suivant...

 Voiture(3,15)

Un autre petit programme en Groovy 2.x

 
                    @groovy.transform.ToString 
                    class Voiture { 
                        Integer nbPorte = 3 
                        String couleur
                    } 
                    
                    @groovy.transform.TypeChecked 
                    def test() { 
                        new Voiture(couleur:15) 
                    } 
                    
                    test()

Le résultat obtenu est une erreur de compilation...

 1 compilation error: [Static type checking] - Cannot assign value
                    of type int to variable of type java.lang.String at line: 9, column: 24

Vérifications et tests unitaires

Qu'est ce qu'un test unitaire ?

Un test unitaire est le test d'une unité de code en isolation.

  • "Unité" - Plus petite partie du code que l'on peut tester en isolation

  • "En isolation" - Séparé de toute autre partie du logiciel, de toute autre unité de code

  • L’environnement opérationnel est simulé si nécessaire par des «faux» objets (mocked objects, stubs)

Caractéristiques cibles pour une unité de code

  • Unité de compilation reconnue par le compilateur

  • Occupe un seul fichier

  • Nombre de lignes de code peu élevé

  • Atomique : ne devrait pas est "découpable" en d'autres unité

En programmation orientée objet, une unité de code est en général une classe.

Focus sur Spock

Spock is a testing and specification framework for Java and Groovy applications. What makes it stand out from the crowd is its beautiful and highly expressive specification language. Thanks to its JUnit runner, Spock is compatible with most IDEs, build tools, and continuous integration servers. Spock is inspired from JUnit, jMock, RSpec, Groovy, Scala, Vulcans, and other fascinating life forms.
